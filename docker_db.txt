postgreSQL inside docker
 
docker-compose.yml creates the postgresql image/container with:
    - user
    - password
    - database_name

docker compose up -d  # start container


3 ways to connect to postgreSQL db: 
    1. from inside docker(without password): 
        docker exec -it licenta_postgres psql -U licenta_user -d licenta_db
    
    2. from outside the docker(with password):
        psql -h localhost -p 5432 -U licenta_user -d licenta_db

    3. from backend(with password): 
        via connecting string using Database_URL 

docker compose down - stop postgreSQL when done for the day*


docker compose down -v # removes the data volume, only when:
    - You want to completely reset the database
    - You messed up and want to start over
    - You're changing PostgreSQL configuration
